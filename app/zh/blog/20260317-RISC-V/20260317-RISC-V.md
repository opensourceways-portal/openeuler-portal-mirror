在 openEuler 24.03 LTS SP3 RISC-V Preview 版本经历充分的社区验证与稳定性加固后，SP3 现已正式移入 LTS 发布目录，进入长期维护阶段。

SP3 由中国科学院软件研究所（简称软件所）智能软件研究中心牵头，联合中兴通讯、阿里巴巴达摩院、算能科技、进迭时空、超睿科技等产业伙伴协同推进，是首个支持 RVA23 标准的长周期维护版本，也实现了面向 Server Platform 规范的初步支持，并将在后续通过 update 持续跟随上游演进。

## 双版本体系支持

自 openEuler 24.03 LTS 实现三架构同源构建以来，社区持续完善面向服务器与开发者场景的 RISC-V 软件栈能力。随着 RISC-V Server Platform Specification 的持续推进，RVA23 已成为新一代高性能 ISA 的主选配置。
本次 SP3 完成对 RVA23 的系统化支持与稳定性验证，并持续提供 RVA20 与 RVA23 双版本支持。在覆盖既有硬件生态的同时，为新一代高性能 RISC-V 平台提供稳定、可持续演进的软件基础。

## RVA23 软件栈能力增强

SP3 围绕 RVA23 软件基线完善了编译工具链与用户态软件栈，涵盖编译器、语言运行时及关键基础库的架构适配与性能优化，相关能力随正式版本纳入长期维护基线。其中，编译工具链方面采用 GCC 14.3.1、Binutils 2.42 以及 LLVM 20 等组件提供 RVA23 架构支持；语言运行时方面完成 Golang 1.21 与 OpenJDK 21 的相关适配与构建修复；基础库方面则对 OpenSSL 与 ISA-L 等关键组件进行了 RISC-V 向量扩展相关优化。同时，还完成了部分用户态软件包的构建适配及基础设施调整，进一步提升了 RISC-V 软件栈的工程成熟度与整体稳定性。

在强化 RVA23 软件栈能力的同时，本次版本也对 RVA20 基线中的关键组件进行了稳定性加固，包括 GCC 12.3.1 的 RISC-V 原子指令相关修复，以及 GRUB 2 在部分平台启动阶段重定位问题的修复。

## RVCK 内核能力支撑

在 openEuler 24.03 LTS SP3 RISC-V 的研发过程中，获得了如意社区 RVCK (RISC-V Common Kernel) [1] 的支持，围绕 RVA23 标准开展内核能力建设，累计合入 931 个补丁，为相关服务器能力的实现奠定了基础。

## SpacemiT K3 平台支持

SP3 完成了对 SpacemiT K3 平台的支持，使其成为首个进入 openEuler LTS 发布体系的 RVA23 SoC。当前版本已支持 SD 卡存储与 GMAC 千兆以太网等关键外设，系统镜像可通过 extlinux.conf 启动，并已在 COM260 开发板上完成验证。后续 LTS 生命周期中，K3 平台相关能力将通过 update 机制持续增强。

## 后续工作

在 LTS 生命周期内，openEuler RISC-V 将持续完善架构能力、内核基础设施与开发生态体系，围绕 RVA23 架构、RVCK 内核能力与 DevStation 开发生态持续推进优化与完善，进一步提升 RISC-V 服务器软件栈的稳定性与可用性。
